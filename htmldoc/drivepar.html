drivepar

PURPOSE

DRIVEPAR Compute Cartesian path drive parameters

SYNOPSIS

function dp = drivepar(t1, t2)

DESCRIPTION 

DRIVEPAR Compute Cartesian path drive parameters

    DP = DRIVEPAR(T0, T1)

 Returns a vector of drive parameters required to plan a Cartesian path 
 from T0 to T1.

 This function has been obsoleted by quaternion-based interpolation in
 TRINTERP.

 Ref:
    Robot manipulators: mathematics, programming and control
    R.P. Paul, MIT Press, 1981.

 See also: CTRAJ, TRINTERP.

0029     % Using the notation of Paul, dp is
0030     %    dp(1)    x
0031     %    dp(2)    y
0032     %    dp(3)    z
0033     %    dp(4)    phi
0034     %    dp(5)    theta
0035     %    dp(6)    psi

0037     dp = zeros(1, 6);
0038 
0039     n1 = t1(1:3,1);
0040     o1 = t1(1:3,2);
0041     a1 = t1(1:3,3);
0042     p1 = t1(1:3,4);
0043     n2 = t2(1:3,1);
0044     o2 = t2(1:3,2);
0045     a2 = t2(1:3,3);
0046     p2 = t2(1:3,4);
0047     
0048     dp(1) = n1'*(p2 - p1);
0049     dp(2) = o1'*(p2 - p1);
0050     dp(3) = a1'*(p2 - p1);
0051 
0052     dp(6) = atan2(o1'*a2, n1'*a2);                % psi
0053     sp = sin(dp(6));
0054     cp = cos(dp(6));
0055 
0056     dp(5) = atan2(sqrt((n1'*a2)^2 + (o1'*a2)^2), a1'*a2);    % theta
0057     st = sin(dp(5));
